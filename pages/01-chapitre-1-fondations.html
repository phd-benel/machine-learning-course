Chapitre 1 : Fondations et Rupture de Paradigme

Objectif : comprendre la bascule du Software 1.0 (règles explicites) vers le Software 2.0 (programmes appris via optimisation), et situer précisément IA / ML / DL dans une hiérarchie conceptuelle cohérente.

Points clés

  • Le paradigme Software 2.0 : l’algorithme devient un modèle paramétrique appris à partir des données.
  • Les 3 piliers : Data (volume/qualité), Compute (matériel + efficacité), Algorithmes (structures + optimisation).
  • Définitions : IA ⊃ ML ⊃ DL, avec des capacités (prédire, classer, percevoir) et des contraintes distinctes.

À retenir : le “code” d’un système apprenant se déplace partiellement des règles écrites par l’humain vers un espace de paramètres ajustés par minimisation d’une fonction de perte sur des données.

Mini-lexique (très court)
  • Données (Data) : observations + labels éventuels, avec biais/qualité/coverage.
  • Modèle : famille de fonctions paramétrées.
  • Optimisation : procédure qui ajuste les paramètres pour réduire une perte.
