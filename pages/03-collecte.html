Chapitre 3 - Phase 1 (3h): La collecte

Objectif: transformer un chaos de fichiers en une base fiable et tracable pour les etapes suivantes.

  • Sources: Inside Airbnb (`listings.csv`, `reviews.csv`) + dossier images.
  • Data Lake: stockage brut, horodate, non destructif.
  • Data Warehouse: structure cible relationnelle pour requetage analytique.
Consignes pratiques

Concevoir une ingestion batch, controler qualite (doublons, nulls, types), logguer les erreurs et conserver les versions de fichiers.

docker compose up -d
docker ps
# verifier les services d'ingestion et de base
